# Code & Languages

Users may develop against Emory's OMOP-CDM with any tool that can access our data lake(s). The following are common languages used by the OHDSI community and locally at Emory.

<div class="grid cards" markdown>

-   :simple-r:{ .lg .middle } **R & RStudio**

    ---

    R is the preferred language of OHDSI developers, with extensive ready-made tooling for observational research. RStudio provides an integrated development environment for data analysis and visualization.

    [:octicons-arrow-right-24: HADES packages](https://ohdsi.github.io/Hades/){ .md-button .md-button--primary }
    [:octicons-arrow-right-24: Emory's R repositories](https://ohdsi-atlas.emory.edu/atlas/#/home){ .md-button }

    [:octicons-arrow-right-24: R training resources](../../Training/Emory/R/index.md)

-   :simple-postgresql:{ .lg .middle } **SQL**

    ---

    Direct queries against Emory's OMOP data lake on Redshift. Use DBeaver, DataGrip, or any SQL client.

    [:octicons-arrow-right-24: Emory query library](../../Training/Emory/SQL/Query%20Library/index.md){ .md-button .md-button--primary }
    [:octicons-arrow-right-24: OHDSI query library](https://data.ohdsi.org/QueryLibrary/){ .md-button }

    [:octicons-arrow-right-24: SQL tips for OMOP](../../Training/Emory/SQL/index.md)
    [:octicons-arrow-right-24: General SQL reference (W3Schools)](https://www.w3schools.com/sql/)

-   :material-language-python:{ .lg .middle } **Python**

    ---

    Increasingly used in the OHDSI community for data analysis, machine learning, and ETL pipelines. Connect using `redshift_connector` or any Redshift-compatible driver.

    [:octicons-arrow-right-24: Database connection guide](../../Support/Access%20Requests/Databases/index.md#after-youre-approved){ .md-button }

</div>
